# LLD Practice - Go

Go implementations of Low-Level Design problems.

## Practice Problems

### Beginner (Use 2-3 patterns)
| Problem | Key Patterns | Focus |
|---------|--------------|-------|
| Parking Lot | Strategy, Factory, State | OOP basics |
| Vending Machine | State, Command | State transitions |
| Logger System | Singleton, Chain of Resp | Thread-safety |

### Intermediate (Use 3-5 patterns)
| Problem | Key Patterns | Focus |
|---------|--------------|-------|
| Elevator System | State, Strategy, Observer | Concurrency |
| Cache System (LRU) | Strategy, Decorator | Data structures |
| Rate Limiter | Strategy, Singleton | Concurrency |

### Advanced
| Problem | Key Patterns | Focus |
|---------|--------------|-------|
| Hotel Booking | Factory, State, Observer | Transactions |
| Task Scheduler | Command, Strategy | Goroutines |
| Message Queue | Observer, Command | Channels |

## Approach (45 min)
1. Clarify requirements (5 min)
2. Identify entities (5 min)
3. Choose patterns (5 min)
4. Design interfaces (10 min)
5. Implement (20 min)
